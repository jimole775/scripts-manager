# 自动点击编辑
## 功能描述
视图展示串行的流程节点图，每个节点表示一个点击操作，节点之间用箭头连接，表示点击操作的顺序。
编辑好之后，点击保存按钮，将流程节点图保存到配置文件中。
并生成一个读取配置文件的脚本，用于在实际运行时读取配置文件并执行自动点击操作。

## 页面布局
0. 上侧是类似tab的标签页，每个标签就是一个脚本的编辑页面，最后一个tab是新增按钮。
1. 左侧是节点模板列表，类型有：图片节点、浏览器dom路径节点、文本输入节点、睡眠节点。
2. 中间是流程节点图的视图区域，用于展示和编辑点击操作的流程，可参考/d/my_work/trae-pros/svelte-flow-demo项目的实现。
3. 右侧是属性面板，具体属性暂定。
4. 两个操作按钮，保存和执行，悬浮在窗口右下角。

## 节点属性
所有节点都是上下布局，上是title、关闭按钮, 下是操作区域，每个节点的操作区域不同。
每个节点都有两个出口连接点，一个表示节点操作成功，一个表示节点操作失败。
1. 图片节点：下是图片上传区域。
2. 浏览器dom路径节点：下是nodePath输入框。
3. 文本输入节点：下是文本输入框，支持随机、手输、固定文本+随机，三种类型。
4. 睡眠节点：下是睡眠时间输入框，支持随机、手输，两种类型，后缀固定单位为秒。

## 操作步骤
1. 初始化一个固定节点，作为流程的起点。
2. 从左侧拖拽节点到中间的流程节点图视图区域，当作新增节点。
3. 使用链接线连接两个节点
4. 重复步骤2和3，添加更多的节点和连接线，组成一个完整的流程节点图。
5. 点击保存，将流程节点图保存到配置文件中。
6. 点击执行，根据配置文件中的流程节点图，执行自动点击操作。

## 技术栈
1. 前端：vue-flow
2. 脚本：根据当前项目情况，自行决定用python还是js实现，性能优先。

## 脚本要求
1. 脚本的自动点击类似于播放，每个节点的点击操作按照顺序执行。
2. 当脚本在执行时，electron app自动最小化显示，保存和执行按钮禁用，直到脚本执行完毕。
3. 脚本必须提供一个停止函数，用于在实际运行时停止自动点击操作，快捷键为Ctrl+C，把快捷键文本描述显示在显示器的左下角。
4. 脚本必须提供一个暂停函数，用于在实际运行时暂停自动点击操作，快捷键为Ctrl+P，把快捷键文本描述显示在显示器的左下角。
5. 脚本必须提供一个继续函数，用于在实际运行时继续自动点击操作，快捷键为Ctrl+R，把快捷键文本描述显示在显示器的左下角。
